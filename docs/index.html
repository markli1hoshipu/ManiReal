ManiReal

A realistic bimanual manipulation benchmark with systematic robustness perturbations.

Code Perturbations Tasks Install Cite

Overview

ManiReal is built on top of the RoboTwin simulation framework and extends it along two axes: (1) a library of visually realistic office, study, and kitchen scenes, and (2) a systematic perturbation suite that applies controlled noise to the language instruction, vision observations, and objects in the scene — enabling per-axis evaluation of policy robustness.

All perturbations are implemented as drop-in YAML configurations. Running the same task with bench_demo_*_clean.yml vs bench_demo_vision.yml produces trajectories from the same success-checker but with visually or semantically shifted observations.

Perturbation axes

Language

Per-episode, the natural-language instruction is re-sampled from a curated bank of paraphrases (~8–10 per task, 25 tasks, ~276 strings total). The selected instruction is written into scene_info.json and made available to the policy via the standard instruction API.

Vision

Three sub-factors, any of which can be toggled independently:

  • Lighting — L1 per-channel diffuse tint in U[0.4, 1.8]; L2 spherical direction with θ ∈ U[8°, 82°], φ ∈ U[0, 2π); L3 50%-chance specular/shininess rewrite (specular ∈ U[0.3, 6.0], shininess ∈ U[10, 256]); L4 50% shadow toggle.
  • Blur — per-episode cycle through five types (motion / gaussian / zoom / fog / glass) at severity 2 (s=0.25), strength-scaled.
  • Pixel shake — per-frame pixel shift dx, dy ∈ U[-max·strength, +max·strength] via cv2.warpAffine.

Object

Three sub-factors applied per episode:

  • Target texture swap — after load_actors, a random PNG from assets/background_texture/seen/ is bound as the base color map of the target object's render body components.
  • Unseen obstacles — obstacle sampling switches from the objects.<scene>.obstacles pool to a visually-disjoint object_ood.<scene>.obstacles pool (targets remain in the baseline pool so the success checker stays consistent).
  • background_plus — B1+ samples a wall/floor color tint; B2 samples table metallic and roughness from configured ranges.

Install & run

git clone https://github.com/markli1hoshipu/ManiReal.git
cd ManiReal
# Follow: https://robotwin-platform.github.io/doc/usage/robotwin-install.html
cd customized_robotwin
source set_env.sh
export ROBOTWIN_BENCH_TASK="bench"

# Collect 3 episodes under a perturbation config:
bash collect_data.sh put_mouse_on_pad bench_demo_vision 0

See install.html for the large-mesh fetch step and the CuRobo cache patch.
